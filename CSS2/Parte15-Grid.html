Gird

Permite trabajar en el layout en base a grillas.
Tenemos que estudiar 6 conceptos.

  • Grid containter: Es como el flex containter. Pero destinado a grillas.
  • Grid item: Es todo lo que este dentro del grid conteinter. Es el hijo del container.
  • Grid cell: Estas son las celdas que estan dentro del item, es hijo directo del grid item, y nieto del conteiner.
  • Grid Tracks (column y row): Son las filas y las columnas. Grid trancks es la suma de filas y columnas.
  • Grid area: Las areas las definimos nosotros, y tienen que ser consecutivas. Entonces conceptualmente, si o si son cuadradas o rectangulas.
  • Grid line (column line y row line): Estas son las linas divisoras de lineas, o columnas.

Propiedades para containers

  • Grid-template-rows: Medida; Esto lo que hace es darle filas a la grilla. Sin esta propiedad, aunque parece que haya filas, se denominan grids implicitos. No se concideran filas todavia.
    Puede haber 9 elementos pero si solo aplicamos a 3 filas esta propiedad, solo va a haber 3 filas.
  • Grid-template-colums: Medida; Y con esto le damos columnas a la grilla, funciona de la misma manera, como en el ejemplo de arriba, damos 3 medidas de rows, 3 medidas para columns y nos queda una grilla completa de 3x3.
    Podemos dar una unidad de medida a ambas propiedades que es 1fr que seria algo parecido al flex-grow. Es dejar que que la columna o fila que tenga esta medida, el espacio sobrante de la pantalla. Se puede dar el fr a varias columnas o filas.
    Podemos colocar una funcion que es repeat(cantidad,parametro de medida) y esto nos evita tener que poner 3 veces 150px por ejemplo. Podemos pasarle otra medida, entonces va a repetir la X cantidad de veces las medidas que les pacemos.
  • grid-gap (O grid-column-gap o grid-row-gap): Es una manera de dar espacio entre las celdas, pero no de las celdas de la grilla.


Propiedades para items

  • grid-row (grid-row-start y grid-row-end):numero / numero Se utiliza para que una celda ocupe mas lugar que otras, colocando la linea donde empieza y la linea donde termina. Pero tambien empueja las siguientes celdas como se ve en el ejemplo.
  • grid-column (grid-column-start y grid-column-end): numero / numero Lo mismo que arriba. Hay que tener cuidado con esto porque se empiezan a descolocar todas las cajas!!!. Y otra variable que podemos pasarle es span
    Un ejemplo para span seria algo: 1 / span 3. Que quiere decir: Empeza en la fila o columna 1 y sumame 3 filas o columnas
