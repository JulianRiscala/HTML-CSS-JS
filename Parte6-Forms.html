Forms

Los formularios HTML son elementos de las páginas web que nos sirven para recuperar información del usuario y enviarla al servidor para que pueda ser procesada.
Fuente

Estructura de forms

Se habre la etiqueta form y siempre lleva un atributo action, usualmente lleva un attribute y un id.

SE PUEDEN SUMAR ATRIBUTOS Y ME DI CUENTA DESPUES DE APRENDER CSS.......... Solo basta con darle un nombre de atributo y listo.

El metodo puede ser uno de dos: get o post.
Con el metodo get, los valores del form son añadidos al final del URL especificado al atributo action.
Ideales para:

  • Fomrs cortos (como "search boxes")
  • Cuando obtienes data del servidor web, y no añadiendola o borrando de la data base

Y con el metodo post, los valores son mandados atraves de lo que conocemos como los HTTP headers.
Ideales para:

  • Permitir al usuario a subir archivos.
  • Contener informacion sensible (como passwords)
  • Añade o elimina informacion de la data base.

Si el form no tiene un method, la data sera mandada usando el metodo get.

El id es usado para identificar el form y distinguirlo por otros elementos de la pagina web.

Text input

La etiqueta input se utiliza para crear distintos tipos de controles, el atributo type determina que tipo de input es creado.
type="text" crea un espacio para una sola linea de texto.

Name

name es el atributo que le indica al servidor que form contiene cada pedazo de data que se ingresa.
Todo control requiere un atributo name.

maxlength

El atributo maxlength setea un numero limite de caracteres que puede tener el control.

type="password" crea un espacio de una sola linea de texto, pero los caracteres son ocultos para el ingreso de contraseñas.
Este tipo de controles no son del todo seguros, no se recomienda utilizar este control. Para full seguridad se recomienda utilizar Secure Sockets Layer (SSL).
De vuelta debemos usar el atributo name, y podemos utilizar el atributo maxlength.

Radio button

type="radio" permite al usuario elegir entre varios elementos.

El atributo name debe ser iguales en todos los radio buttons, este es el atributo que viaja al servidor con el value que es asignado por el usuario.

value es el valor asignado que viaja al servidor, es la opcion elegida. Lo que ve el usuario.

checked es el atributo para indicar que opcion esta indicada predeterminadamente. SI O SI UNO DEBE SER ELEGIDO.

Labels para controles

Cada form deberia tener su propio label para mantener un orden sobre cualquier accion.
En el siguiente ejemplo se utiliza para dos cosas

  • Envolver tanto la descripcion del texto como la entrada del formulario.
  • Mantener una separacion del control form, y utilizar la etiqueta for que a que form se refiere.

for

El for determina que form control esta atado al label. Notamos como los radio buttons usan un id. El valor del id
identifica el elemntos de todos los demas que esten en la pagina web.
El value del atributo for es igual id del form control al que esta "labeado". Esta tecnica puede utilizarse en cualquier form control.
Cuando un label es utilizado con un checkbox o un radio button, el usuario puede hacer click tanto al form como al label y esto le da facilidad al usuario para seleccionar la opcion (por tener un area mas grande).
La posicion del label es muy importante!

Estos son los mejores lugares donde colocar labels dentro del form:

  • Encima o por la izquierda
    • Text inputs.
    • Text areas.
    • Select boxes.
    • file uploads
  • Por la derecha
    • Checkboxes individuales
    • radio buttons individuales
